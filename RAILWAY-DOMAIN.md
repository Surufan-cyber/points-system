# 🚂 Railway 获取访问地址详细指南

## 方法一：在项目概览页面查找（最简单）

1. 在 Railway 项目页面，点击顶部的项目名称（回到概览页面）
2. 在页面顶部应该能看到一个 **"View Logs"** 按钮旁边
3. 如果部署成功，会显示一个类似这样的地址：
   ```
   https://points-system-production-xxxx.up.railway.app
   ```
4. 直接点击这个地址就能访问

## 方法二：通过 Settings 查找

### 新版 Railway 界面：

1. 在项目页面，点击左侧菜单的 **"Settings"**（齿轮图标）
2. 向下滚动找到 **"Networking"** 或 **"Domains"** 部分
3. 点击 **"Generate Domain"**
4. 等待几秒钟，会显示生成的地址

### 如果找不到 Settings：

1. 点击项目名称回到主页面
2. 找到部署的服务（通常显示为 "web" 或项目名）
3. 点击服务名称进入服务详情
4. 在服务详情页找到 **"Settings"** 标签
5. 查找 **"Domains"** 或 **"Public Domain"** 部分

## 方法三：手动添加端口配置

如果上面方法都找不到，可能需要配置端口：

1. 在项目页面点击 **"Settings"**
2. 找到 **"Port"** 设置
3. 输入：`5000`（后端运行的端口）
4. 点击 **"Save"**
5. 然后应该会出现 **"Generate Domain"** 按钮

## 方法四：查看部署日志

1. 在项目页面点击 **"View Logs"** 或 **"Deployments"**
2. 查看最新的部署日志
3. 如果部署成功，日志最后会显示：
   ```
   🚀 服务器运行在端口 5000
   📱 访问地址: http://localhost:5000
   ```
4. 回到项目概览，应该能看到访问地址

## 常见问题

### Q1: 部署失败怎么办？

查看部署日志，常见错误：
- **构建失败**：检查 `package.json` 的 scripts 配置
- **启动失败**：检查环境变量是否设置
- **端口错误**：确保设置了正确的端口 5000

### Q2: 找不到 Generate Domain 按钮

可能原因：
1. 部署还未完成（等待 3-5 分钟）
2. 需要先配置端口（方法三）
3. 服务类型设置错误

解决方案：
```bash
# 在 Railway 项目中添加变量
PORT=5000
```

### Q3: 访问地址显示 404

可能原因：
- 前端构建失败
- 静态文件路径错误

检查步骤：
1. 查看 Railway 构建日志
2. 确认 `npm run build` 执行成功
3. 检查 `server/index.js` 中的静态文件路径
